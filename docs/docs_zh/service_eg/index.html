Service 示例

#概述

在本面文档里，将列举一些已实现的 Service 示例。包括：

  • Metadata Service：支持链的运营方在起链前对链的相关信息进行配置。
  • Asset Service：支持用户发行自定义资产，支持转账，查询等操作。
  • RISC-V Service：支持用户用 c 语言进行合约的开发。
  • Node Manager Service：支持动态添加、删除节点。

#Metadata Service

Metadata Service 负责存储链的元数据信息，为起链所必须配置的服务，以支持链的运营方在起链前对链的相关信息进行配置。

详细信息和接口请参考Metadata Service 文档。

#Asset Service

Asset Service 是一个资产管理服务，用来支持用户发行原生资产，并且管理这些原生资产。

  • 详细信息和接口请参考Asset Service 文档
  • 源代码

#RISC-V Service

RISC-V Service是一个基于 CKB-VM 开发的虚拟机服务。

该服务内置了一个 RISC-V 指令集解释器作为虚拟机。通过该服务，用户可以自由的部署和调用合约，实现强大的自定义功能。 任何支持 RV64I 的编译器 (如 riscv-gcc, riscv-llvm, Rust) 生成的可执行文件均可以作为合约使用。目前该模型支持用户用 C 语言编写合约，后面将支持更多语言。

  • 详细信息和接口请参考RISC-V Service 文档
  • 源代码

#Node Manager Service

Node Manager Service 是负责变更节点的共识配置，并对变更权限进行管理的服务。这些信息存储在 Metadata Service 中，在 Metadata 中可以动态变更的字段有 interval、verifier_list、 propose_ratio、 prevote_ratio、precommit_ratio、brake_ratio 。只有 admin 账户有权限进行变更操作，admin 账户的初始值写在 config/genesis.toml 配置文件中，起链后可以发交易给 Node Manager Service 进行修改。

  • 详细信息和接口参考Node Manager Service 文档。
